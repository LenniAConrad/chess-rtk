# Signature ChessRTK Logo Concepts

This round is intentionally less literal. The goal is to find a distinctive
identity that can become recognizable on its own: state lattices, encoded
strands, proof loops, command sigils, record weaves, and transposition knots.

| File | Concept | Core idea |
| --- | --- | --- |
| `01-state-lattice.svg` | State Lattice | A reproducible position graph with one verified path. |
| `02-fen-helix.svg` | FEN Helix | Encoded chess state as a living, repeatable strand. |
| `03-proof-loop.svg` | Proof Loop | A closed command loop with a checkable result. |
| `04-kernel-orbit.svg` | Kernel Orbit | One rules core with tools orbiting around it. |
| `05-search-prism.svg` | Search Prism | A position split into candidate lines and resolved. |
| `06-command-sigil.svg` | Command Sigil | A terminal mark built from prompt, route, and result. |
| `07-record-weave.svg` | Record Weave | Analysis records woven into datasets and books. |
| `08-engine-pulse.svg` | Engine Pulse | Evaluation as a signal with a stable center line. |
| `09-transposition-knot.svg` | Transposition Knot | Different paths meeting at the same state. |
| `10-rtk-crest.svg` | RTK Crest | A compact identity mark for releases and favicons. |
